# mini4x — 개발 하네스

판타지 미니 4X의 **단일 소스 전투 엔진 + 자동 검증 + 프로토타입 빌드** 도구 모음.

## 왜

전투 엔진이 여러 곳(파이썬 sim, 검증용 JS, 프로토타입 HTML)에 복제돼 있으면,
한 곳에서 수치를 바꿀 때 나머지가 어긋난다("드리프트"). 이 하네스는 엔진을 **한 파일**로 모으고,
그 하나만 고치면 검증과 프로토타입이 함께 따라오게 한다.

## 구성

| 파일 | 역할 |
|------|------|
| `engine.js` | **단일 캔노니컬 전투 엔진** (node + 브라우저 공용). 기획서 §7 규칙 구현. **수치는 여기서만 고친다.** |
| `verify.js` | **자동 밸런스 검증 하네스.** 삼각·지배유닛·카운터·전열분산·티어를 한 번에 검사. |
| `build.js` | `engine.js`를 HTML 템플릿의 `//__ENGINE__` 자리에 인라인 → 자체 완결 프로토타입 생성. |
| `app_combat.html` | Phase A 전투 프로토타입 **템플릿**(UI만, 엔진 자리엔 마커). |

## 워크플로

```bash
# 1) 밸런스 수치를 engine.js 에서 수정

# 2) 삼각·카운터가 유지되는지 즉시 검증
node verify.js          # → 12 PASS / 0 FAIL 이어야 함

# 3) 프로토타입 재생성 (엔진 자동 인라인)
node build.js           # → ../phaseA_prototype.html 갱신
```

`engine.js` 한 곳만 바꾸면 `verify.js`(검증)와 `build.js`(프로토타입)가 같은 소스를 쓴다.

## 검증 항목 (verify.js)

1. **병과 삼각** — 보병군>기병군>궁병군>보병군 순환.
2. **지배 유닛 없음** — 모든 유닛이 최소 1개 카운터 보유.
3. **적응형 편성 보상** — 기병 적에게 창병 투입 시 승률 급상승.
4. **전열 피해 분산(②)** — 보호받는 창병도 유의미하게 피해(공짜 요격 방지).
5. **티어 효과** — 고티어가 저티어 압도.

## 참고 (역사적 산출물, 이제 engine.js가 대체)

`combat_sim3.py`, `combat_cost.py`, `combat_modes.py`, `combat_rounds.py` 등 파이썬 시뮬은
밸런스 탐색·튜닝 기록. 이후 검증은 `verify.js`(engine.js 기반)로 일원화.
