# mini4x — 개발 하네스

판타지 미니 4X의 **단일 소스 로직 + 자동 검증/테스트 + 프로토타입 빌드** 도구 모음.

📄 **게임 기획서: [`docs/기획서.md`](docs/기획서.md)** (v2.0) — 컨셉·시스템 설계·전투 수치(§7)·군 규모(§9)·로드맵. 코드가 인용하는 `§7`/`§9`의 출처. ⚠ §7·§9에 현재 문서↔코드 드리프트(상성 삼각 붕괴 등) 정정이 기록돼 있으니 밸런스 작업 전 반드시 확인.

## 왜

로직이 여러 곳(검증용 JS, 시뮬, 프로토타입 HTML)에 복제돼 있으면, 한 곳에서 바꿀 때
나머지가 어긋난다("드리프트"). 이 하네스는 **전투 엔진·게임 로직·UI를 각각 한 파일**로 모으고,
그 하나만 고치면 검증·시뮬·프로토타입이 함께 따라오게 한다.

## 구성 (단일 소스)

| 파일 | 역할 |
|------|------|
| `engine.js` | **전투 엔진** (node + 브라우저 공용). 3병과×2클래스, 3열, 특수·전열분산·티어. **전투 수치는 여기서만.** |
| `game.js`   | **게임 로직** (맵·경제·건물·영웅·연구·몬스터·AI·턴). `setMap()`으로 노드↔타일 맵 교체. **규칙은 여기서만.** |
| `ui.js`     | **공유 UI** (자원바·성 내정 패널·이동/전투/저장/턴). 노드판·타일판이 이 하나를 공유. **UI는 여기서만.** |
| `verify.js` | **밸런스 자동검증** — 삼각·지배유닛·카운터·전열분산·티어 (12 PASS). |
| `sim.js`    | **400판 플레이테스트 봇** — 승률·게임 길이·자원 잉여 집계로 밸런스 감. |
| `test.js`   | **자동 테스트(브라우저)** — 빌드된 두 판의 로드·전투·턴·저장을 확인(회귀 방지). |
| `build.js`  | 엔진+게임+UI를 템플릿의 `//__ENGINE__`/`//__GAME__`/`//__UI__` 자리에 인라인. |
| `app_combat/economy/tilemap.html` | 프로토타입 **템플릿**(맵별 head + UI 마커). |

## 워크플로

```bash
node verify.js   # 밸런스 12검증 (전투 수치 바꿨을 때)
node sim.js      # 400판 플레이테스트 (승률·페이싱 감)
node build.js    # ../phaseA/B/C 프로토타입 재생성 (엔진·게임·UI 인라인)
node test.js     # 빌드된 두 판을 브라우저로 자동 확인 (반드시 통과)
```

**규칙: 커밋 전 `node build.js && node test.js` 를 통과시킨다.** (전투가 깨졌던 snapshot 이름
충돌 같은 버그를 test.js 가 즉시 잡는다.)

## 맵 교체 (노드 ↔ 타일)

`game.js`는 맵을 `setMap({NODES,EDGES,MONSTERS|spawn})`로 주입받는다. 로직은 맵과 무관.
- 노드판: `game.js`의 `DEFAULT_MAP`(17노드, 고정 둥지).
- 타일판: `app_tilemap.html`의 `buildTileMap()`(14×9 격자, 대칭 자원, 랜덤 몬스터 스폰) → `Game.setMap()`.
