# 판타지 미니 4X — 게임 기획서 v0.1

> 정복이 주인공인, 개인 개발용 소형 4X 전략 게임
> (한 판 30분~1시간 · 싱글플레이)

---

## 1. 한 줄 컨셉

작은 판타지 대륙에서, **상성 좋은 병과와 영웅을 조합해 소수의 적 세력을 완전히 정복**하는 턴제 전략 게임.

## 2. 확정된 뼈대 (함께 결정한 것)

| 항목 | 결정 | 이유 |
|------|------|------|
| 장르 | 미니 4X | 4X 재미는 살리되 범위를 좁혀 완성 가능하게 |
| 테마 | 판타지 | 종족·마법·몬스터로 개성 뽑기 좋음 |
| 핵심 기둥 | **정복 (Exterminate)** | "가장 재미있는 순간"을 전투 승리로 |
| 전투 방식 | 자동 전투 | 구현 부담↓, 재미는 전투 직전 결정으로 이동 |
| 전투 깊이 | 병과 상성 + 영웅·지휘관 | 편성·배치가 승부를 가르는 두 레버 |
| 승리 조건 | 완전 정복 **+ 중앙 레이드** | 정복 정공법 + 조기 종결·역전 루트 (F 참조) |
| 규모 | 소형 (적 2~3세력, 작은 맵) | 스노우볼·개발 난이도 최소화 |

## 3. 코어 루프 (플레이어가 반복하는 것)

1. **정찰/탐험** — 안개 걷힌 맵에서 적 위치·중립 거점 파악
2. **생산** — 도시에서 병과 뽑고 영웅 배치
3. **편성** — 상대 병과에 유리한 조합 + 영웅 스킬 매칭
4. **진군 → 자동 전투** — 결과를 보고 다음 전선 판단
5. 반복하며 적 세력을 하나씩 제거 → 완전 정복

> 핵심 질문은 항상 하나: **"어떤 영웅에게 어떤 병과를 붙여, 상성 좋은 곳에 보낼까?"**

## 4. 시스템 설계

### A. 자원·경제 — **확정: 4자원 (식량·목재·석재·철)**

역할이 겹치지 않게 담당을 분리:

| 자원 | 담당 | 성격 |
|------|------|------|
| 식량 | 군대 유지·보급 (규모 상한) | 생산 비용 + 유지비 |
| 목재 | 초급 유닛·기본 건물 | 확장 초반 주력 |
| 석재 | 성벽·방어 시설·상급 건물 | 인프라/방어 전용 |
| 철 | 정예·상급 유닛의 핵심 재료 | **전투력 병목 자원** |

**소모 규칙**
- 병력 **생산** = 식량 + 목재 + 철 (고티어로 갈수록 철 요구량 증가)
- 병력 **유지** = 식량 (보유만 해도 매 턴 지속 소모)
- **건물** = 목재 + 석재 + 철

**설계 의도**
- 철이 유닛·건물 양쪽에 필요 → 진짜 병목이 되어 "병력이냐 건물이냐" 매 턴 긴장.
- 식량이 생산비 + 유지비 이중 역할 → 대군 남발을 자연스럽게 억제(군대 규모 자율 조절).
- 석재는 건물 전용 → 방어·인프라 색깔이 뚜렷.

### B. 종족·세력 — **확정: 전 세력 인간 통일**

- 종족 상성이 없는 대신 **영웅과 편성이 세력의 개성**을 만든다 (핵심 깊이 레버와 일치).
- 세력 간 차별화는 영웅 라인업 중심으로 (+ 시작 지형·초기 보너스는 추후 결정).

### C. 병과·전투 해결 — **확정: 3병과 × 2클래스 (6유닛)**

**전투 모델**
- 라인 기반 자동 전투. 모든 유닛의 **기본 공격은 앞라인 우선**.
- **배치는 자동 (3열 고정)**: **보병(앞) — 기병(중) — 궁병(뒤)**.
- 각 유닛의 **특수 능력은 확률 발동** → 대상 없으면 앞라인을 타격(폴백).
- 삼각 구도: **보병 > 기병 > 궁병 > 보병** (창병=대기병, 경기병=대궁병, 궁병=대보병).

**데미지 분배 규칙** (개념 — 검증된 수치는 §7)
- 특수 발동(대상 존재) → 두 표적 열에 **분산** (§7: 각 0.6배, 명중 시 총 120%).
- 특수 대상 없음(폴백) → 앞라인에 **집중** (§7: ×0.85, 헛맞춤에 소폭 페널티).
- 특화 유닛(석궁병·중기병)은 보병(앞)만 노려 **풀 데미지 집중** → 앞라인 파괴 경주에서 우위.
- 범용 유닛: 장궁병 = 앞+중(보병+기병), 경기병 = 앞+뒤(보병+궁병).

> 효과: "앞라인 빨리 뚫기(특화) vs 뒷라인 갉기(범용)" 긴장이 수치에서 자연 발생. 별도 밸런스 조정 부담↓.

| 병과 | 클래스 | 역할 | 특수 능력 |
|------|--------|------|-----------|
| 보병 | 중갑보병 | 방어 탱커·앞라인 유지 | 높은 방어력 |
| 보병 | 창병 | 대기병 | 기병 공격 시 **확률로 카운터** |
| 궁병 | 장궁병 | 범용 딜러(뒷라인) | 확률로 **보병+기병 동시** 타격 |
| 궁병 | 석궁병 | 대보병 특화(뒷라인) | 보병만 공격, 대신 **대보병 딜↑** |
| 기병 | 경기병 | 범용 기동 | 확률로 **보병+궁병 동시** 타격 |
| 기병 | 중기병 | 대보병 돌격 | 보병만, **대보병 딜↑** (창병 카운터에 노출) |

**밸런싱 원칙**
- 특화 유닛(석궁병·중기병)은 자기 표적(보병)에서 범용 유닛보다 **데미지 우위**여야 존재 가치가 생김("넓게 vs 깊게" 교환).
- 특수가 **의도 대상 명중 시 > 앞라인 폴백** (예: 뒷라인 명중 시 보너스 딜) → 상성 맞추는 동기 유지.
- 보병이 순수 고기방패가 안 되도록 **공격 가치**도 부여(예: 앞에 붙은 돌격 유닛 확실히 처치).

### D. 영웅 — **확정: 버프형 지휘관, 내정/전투 2계열**

- **1부대 = 1지휘관** (추후 다인 편성으로 확장 여지).
- 영웅은 **직접 전투 없이 버프만** 제공 → 자동 전투 단순함 유지.

| 계열 | 성에 배치 | 부대에 배치 |
|------|-----------|-------------|
| **내정 영웅** | 건축 속도·연구 속도 ↑ | (채집 부대) 채집 속도·채집량 ↑ |
| **전투 영웅** | 수비 버프 (공/방/체 ↑) | 공격 버프 (공/방/체 ↑) |

**설계 의도**
- 내정 vs 전투 배분 = 경제의 "성장 vs 전쟁" 긴장과 겹침 → 템포 결정.
- 전투 영웅의 성/부대 대칭 → "수도를 지킬까 원정을 갈까"가 매번 선택.
- 1부대 1지휘관 → 채집 부대는 전투 버프 포기 = 몬스터 도는 맵에서 호위 리스크.

**획득 사다리**
- **스타터 영웅** 최초 제공.
- **선술집(성 내)**: 주기적으로 랜덤 영웅 등장 → **등급 캡**(고등급 안 나옴) + **영입 시 재화 소모**로 이중 희소성.
- **토벌 점수**: 토벌로 점수 획득 → 소모해 선술집에서 **고등급·특별 영웅** 영입.

**성장**
- **경험치 아이템**으로 통일 (내정 영웅 기여도 측정 문제 회피).
- XP 아이템은 **사냥·토벌 보상**으로 획득 → 몬스터 콘텐츠가 성장까지 이어줌.
- "어느 영웅에게 먹일까"라는 **배분 결정** = 게임 전체 DNA와 일치.

**밸런싱 원칙**
- 영웅은 슬롯보다 **희소**해야 배치 퍼즐이 성립 (로스터 상한 고려).
- **재배치에 이동 시간/비용** 부여 → 매 턴 셔플 방지, 배치를 커밋으로.

**결정 중**: 성의 영웅 슬롯 수, XP 아이템이 계열 공통인지 구분인지.

### E. 몬스터·PvE — **확정: 사냥 / 토벌 2티어, 보상 전용, 개별 개체**

- 몬스터는 **보상 전용** — 포획·영입 없음, 영토 확장과도 무관한 순수 도전 콘텐츠.
- 맵 곳곳에 중립 위협으로 서식. **✅ 리스폰 구현**: 둥지 처치 후 8턴 뒤 재생성 → 사냥·토벌이 반복 수입·경험치·토벌 점수원(성장 시스템의 연료). 레이드(고대 생물)는 미리스폰(승리 직결).
- **✅ 맵 확장**: 12→17노드. 사냥 둥지 2(늑대·도적), 토벌 둥지 2(고블린·오크 진지), 레이드 1(고대성). **침공 회랑(P→E, 평지·자원, 몬스터 無)** 확보 + 둥지는 가지에 배치(선택적 사냥). 확장 맵 대응 이동력 +1(기본 5).
- **몬스터는 부대 편성이 아닌 개별 개체** — 각자 고유 스탯·행동 (베스티어리 느낌).
- **개체도 앞/뒤 위치를 가짐** → 플레이어의 3열·특수 규칙이 그대로 적용.
  - 이유: 위치가 없으면 범용 유닛(경기병·장궁병)의 특수가 늘 폴백(8~9) → PvE에서 특화 유닛에 일방적으로 밀림. 위치를 줘야 6유닛이 PvE·PvP 양쪽에서 제 몫.
- **토벌은 "다중 위협"으로 설계** (단일 HP통이면 특화 몰빵이 정답이 되어 조합이 무의미):
  - 고체력 + 근접 강타 + 원거리 공격을 겸비하거나, 역할이 다른 개체 무리(탱커형+원거리형).

| 티어 | 필요 전력 | 성격 | 보상 |
|------|-----------|------|------|
| **사냥** | 기병 등 기동 유닛 소수 (약한 기동형 개체) | 빠른 반복, 초반 성장 무대 | 식량·재료 (소소) |
| **토벌** | 병과 조합 + 영웅 | 난이도 높은 도전 | 유물·상급 재료·대량 자원·영웅 경험치 |

**설계 의도**
- PvE가 PvP와 같은 근육(사냥=기동, 토벌=병과 조합)을 훈련 → 콘텐츠가 코어와 한 몸.
- 정복 전까지 "할 일"이 계속 생겨 스노우볼 지루함 완화.

### F. 승리 조건·페이싱 — **확정: 이중 승리 + 중앙 레이드**

**승리 조건 2종**
1. **완전 정복** — 모든 적 세력 제거.
2. **중앙 레이드** — 맵 중앙 고대성을 수호하는 **고대 생물 처치 → 성 점령 → 수성 N턴(3~5) 유지 시 승리**.

**페이싱 효과 (스노우볼 방지)**
- 앞선 자: 청소 노동 대신 중앙 점령으로 **조기 종결**.
- 뒤처진 자: 정복 열세여도 레이드로 **역전 루트**.
- 중앙은 모두가 노리는 화약고 → 레이스 긴장(AI도 도전), 수성 중 본토가 비는 전략적 대가.

**밸런싱 원칙**
- 고대 생물 = **엔드게임급 토벌**(제대로 된 조합 + 성장한 영웅 필수) → 성장·전쟁 건너뛴 러시 방지.
- 수성 N턴은 짧게(3~5), 고대성에 방어 보정 → 긴장은 살리되 판 늘어짐 방지.
- 이미 깔린 브레이크: 식량 유지비(군대 규모 억제) + 소형 맵(짧은 판).

### G. 성장·턴 구조 — *기본 채택(추후 상세)*
- **테크(티어)**: 병과 티어 해금(기본 T1 → 정예 T2 → 상급 T3). — ✅ **구현: 생산 건물(병영·궁수대·마구간) 레벨업이 해당 병종의 생산 가능 최대 티어를 게이트.** 고티어 = 스탯 배율↑(engine.tierMult) + 비용↑(특히 철). 저장은 복합 키 `이름@T2`.
- **턴 구조**: 클래식 턴제. 한 턴 = 이동 → 생산 지시 → 전투 자동 처리 → 적 턴. — *잠정*

## 6. MVP 범위 (프로토타입 목표)

### 검증할 핵심 가정
> **"자동 전투에서 병과 상성 + 영웅으로 군대를 짜는 결정이 재미있는가?"**

자동 전투는 플레이어가 전투를 직접 조작하지 않으므로, 재미가 전부 **전투 직전 편성 결정**에서 나와야 함. 이게 무너지면 게임 전체가 무너짐 → **가장 먼저·가장 싸게** 검증한다.

### Phase A — 전투 검증 프로토타입 *(먼저, 최소)*
목표: **전투 한 판이 재밌는지만** 본다. 4X 요소 없음, 플레이스홀더 그래픽.

포함:
- 6유닛(3병과 × 2클래스), 3열 자동 배치
- 자동 전투 해결: 기본공격 앞라인 + 특수 확률 발동 + 데미지 분배(5/5, 폴백 8~9)
- 양쪽 군대 **수동 편성**(적은 프리셋) → 실행 → 결과 관전
- 전투 영웅 버프 1종 토글

검증 질문: 상성이 눈에 읽히나? 카운터 짜는 결정이 만족스러운가? 관전이 지루하지 않나?

### Phase B — 수직 슬라이스 MVP *(전투가 재밌다고 확인된 뒤)*
목표: **최소한의 완결된 게임 한 판** (성장 → 편성 → 정복 루프 1바퀴).

포함:
- 소형 맵: 내 성 1 + 적 성 1 + 자원 노드 몇 + 사냥 몬스터 2~3
- 4자원 채집(유닛/영웅 배치)
- 성에서 6유닛 생산 (테크 없이 전부 해금)
- 영웅: 스타터 내정 1 + 전투 1 (선술집·성장 없이 고정)
- 사냥 몬스터 몇 마리 (PvE 맛보기)
- AI 1세력 (아주 단순: 생산 + 진격)
- 승리: **완전 정복만** (적 성 함락)

검증 질문: 한 판이 처음부터 끝까지 재밌게 굴러가나?

### MVP에서 빼는 것 (나중에)
- ~~선술집 랜덤 영웅 + 재화 영입, 토벌 + 토벌 점수 영입~~ ✅ **구현**: 선술집(건물) → 주기적 랜덤 후보(계열·등급 랜덤, ★1~2) 재화 영입 · **토벌/레이드 처치 → 토벌 점수** → ★3 특별 영입. 영웅 **등급(★1~3)이 버프 세기 결정**(전투 +15/20/28%, 내정 채집·생산 차등).
- ~~중앙 레이드 승리(고대성·고대 생물)~~ ✅ **구현**: 중앙 `고대성` 노드 + 고대 생물(레이드급 보스). 처치 → 점령 → **4턴 수성** 시 승리. 점령 중 방어 +20%, 비우면 진행 리셋. **AI도 레이드 도전**(대군 편성 시 고대성 진격).
- ~~테크 3티어~~ ✅ **구현**(위 G 참조) · 다수 적 세력
- 영웅 XP 아이템 성장·재배치 이동, 페이싱 장치(약자 연합 등)
- 아트·UI 폴리시 (플레이스홀더로)

### 왜 이렇게 자르나
- **리스크 큰 것부터**: 자동 전투 재미(A) → 전체 루프(B).
- 토벌·레이드·테크는 "재미의 **확장**"이지 "재미의 **증명**"이 아님 → 코어 검증 후 추가.
- ⚠️ **AI가 숨은 난관**: 똑똑한 AI는 비쌈 → MVP에선 스크립트형 단순 AI로 충분.

## 7. 전투 수치·계산 (**시뮬레이션 검증 완료** — v3 튜닝)

### 스탯 종류 (4종 + 병력수)
- **HP** 체력 · **ATK** 공격력 · **DEF** 방어력 · **Proc%** 특수 발동 확률
- **병력수(N)**: 한 슬롯 = 같은 유닛 N명의 스택. 스택 총 HP = N × 개체 HP, 라운드 공격 횟수 = 살아있는 개체 수.

### 유닛 기본 수치 (개체 1명 기준) — 검증된 값

| 유닛 | HP | ATK | DEF | Proc% | 특성 |
|------|----|----|----|------|------|
| 중갑보병 | 120 | 8 | 12 | — | 순수 방어 탱커 (특수 없음) |
| 창병 | 100 | 12 | 6 | 55% | 전열 요격(대기병 반격) |
| 장궁병 | 65 | 16 | 3 | 35% | 발동 시 **앞+중(보병+기병)** 배분 |
| 석궁병 | 60 | 16 | 3 | — | 보병만 타격, **대보병 ×1.4** |
| 경기병 | 95 | 17 | 5 | 45% | 발동 시 **앞+뒤(보병+궁병)** 배분 |
| 중기병 | 105 | 12 | 8 | — | 보병만, **대보병 ×1.4** (창병 요격에 노출) |

**전역 상수**: 칩 피해 하한 15% · 폴백 계수 0.85 · 특화 배수 ×1.4 · 특수 분배 각 표적 **0.6**(명중 시 총 120% = 연결 보상) · 창병 요격 배수 ×2.0

### 전투 계산 규칙 (라운드제 자동)
1. **배치**: 양측 3열 — 보병(앞) · 기병(중) · 궁병(뒤).
2. **라운드 진행**: 매 라운드 각 스택의 살아있는 개체가 1회 공격.
3. **타겟팅**: Proc% 굴림 → 발동 시 특수 타겟 열(**대상 열 없으면 폴백** 앞라인 ×0.85), 미발동 시 기본(앞라인).
4. **공격력 배분**: 특수 발동 시 두 표적 열에 각 **0.6배**씩. 기본/폴백은 앞라인 집중.
5. **피해 공식**: `개체피해 = max(배분ATK × 0.15, 배분ATK − 표적DEF)` → 방어력 높아도 최소 칩 피해. 사상자 = 누적피해 ÷ 개체HP.
6. **전열 피해 분산(②, 확정)**: 앞라인(front)을 향한 피해는 앞열의 **모든 스택에 병력수 비례로 분산**. → 중갑 뒤 창병도 피해를 나눠 받음(공짜 요격 방지). *DEF는 명목 표적 기준으로 계산 후 총량을 분산.*
7. **특화 보정**: 석궁병·중기병은 보병 대상 ATK **×1.4**.
8. **창병 요격(핵심 규칙)**: 기병이 상대 **전열(front)** 을 때릴 때, 그 전열에 **창병이 있으면** 창병이 요격 반격(×2.0). *전열 단위 판정.*
9. **후방 노출**: 앞라인 스택 전멸 → 다음 열이 앞라인.
10. **종료**: 한쪽 전멸.

### 영웅 버프 적용
- 전투 영웅: 개체 스탯에 **% 버프**(공/방/체 +x%)를 스택 집계 **전에** 적용.

### ✅ 검증 결과 (시뮬레이터 3회 튜닝, 각 매치업 800회)
- **병과 삼각 순환 성립**: 보병군>기병군(1.00) · 기병군>궁병군(0.72) · 궁병군>보병군(1.00).
- **적응형 편성 보상**: 기병 위주 적에게 창병 투입 시 승률 **0.01 → 1.00**.
- **지배 유닛 없음**: 모든 유닛이 명확한 카운터 보유 (예: 중기병은 창병·경기병에 패배).
- 남은 미세 튜닝(구조적 아님, 플레이테스트 대상): 장궁병은 단독 듀얼에 약함(지원 유닛 성격), 경기병의 PvE 성능 보정.
- 시뮬레이터 코드: `combat_sim3.py` (규칙 그대로 구현).

### ✅ 추가 검증 — "공짜 요격" 문제 해결 (전열 피해 분산 채택)
- **발견**: 중갑이 앞을 막으면 창병이 피해를 안 받고 요격만 넣는 문제. 지속 교전에서 요격/피해 비율 **5.5**(거의 공짜).
- **비교 검증**(`combat_modes.py`): ②전열 분산 vs ①교전(frontage) 모델.
  - ②분산: 비율 5.5→**2.0**(창병도 피해 부담) + **삼각 유지**(기병>궁병 0.67). → **채택**.
  - ①교전: 공짜 요격은 고치나 **삼각 붕괴**(기병>궁병 0.21, 전열 폭이 기병 군단 출력을 깎음). → 기각.
- **정정**: 스팸 문제는 전투 층이 아니라 **§9 슬롯 상한**이 해결(같은 슬롯이면 물량 불가). 교전 모델은 불필요.

> 이 수치는 삼각·카운터가 성립하는 **검증된 출발점**. 세부 밸런스는 실제 플레이테스트로 계속 조정.

## 8. 생산 비용·병력수 (v1 — 전선 폭 결정과 함께 확정 예정)

### 생산 비용 (개체 1명당) + 유지

| 유닛 | 식량 | 목재 | 철 | 유지(식량/턴) | 성격 |
|------|:--:|:--:|:--:|:--:|------|
| 중갑보병 | 2 | 2 | 2 | 1 | 방어 인프라 |
| 창병 | 2 | 3 | 2 | 1 | **싼 대기병**(카운터 접근성↑) |
| 장궁병 | 1 | 3 | 1 | 1 | 최저가 지원 |
| 석궁병 | 1 | 2 | 2 | 1 | 대보병 |
| 경기병 | 4 | 2 | 3 | 2 | 고급 기동(식량·유지↑) |
| 중기병 | 3 | 1 | 4 | 2 | **철 집약 정예** |

**설계 의도**
- **철 = 병목**: 강하거나 무거운 유닛일수록 철↑ ("고티어 철↑" 규칙 준수). 중기병이 철 최다.
- **목재 = 기본**: 창·장궁 등 초반 스팸 유닛이 목재 위주 → 값싸게 전선 확보. 특히 대기병 답인 창병이 싸서 **카운터 접근성**이 좋음.
- **식량 유지**: 기병(말)은 유지 2 → 대규모 기병은 식량 압박 → 자연스러운 규모 억제.

### 병력수 (군대 규모)
- 스택 = 같은 유닛 여러 명. 고정 상한 없음, 대신 **식량 유지비가 실질 상한**: 유지 가능 총 병력 = 식량 잉여 ÷ 평균 유지비.
  - 예) 식량 잉여 +40/턴 → 보병 약 40명 또는 기병 약 20명 유지.
- 생산은 개체(또는 소규모 분대) 단위 큐.

### ⚠️ 검증에서 드러난 구조적 발견 — "전선 폭(frontage)"
- 같은 **병력수** 밸런스는 검증됨(§7). 그러나 같은 **비용**으로 붙이면 삼각이 깨짐 — 싸고 많이 뽑는 유닛(스팸)이 유리.
- 원인: 현재 모델은 스택의 **모든 병력이 매 라운드 공격 + 피격** → 물량이 공격력·생존력 양쪽으로 선형 증가 → 질보다 양.
- **권장 해법 — 전선 폭 도입**: 스택당 라운드에 **교전·전사하는 병력을 K명(예: 10)으로 제한**, 나머지는 예비(교대 지속력만 제공). → 질이 승부를 가르고, 같은 비용 밸런스 복원 + 배치·규모에 의미 부여.
- **→ 해결책 확정: §9 "군 규모 시스템(슬롯 상한)".** 병력수를 캡으로 묶으면 §7의 검증된 밸런스가 작동.
- 검증 코드: `combat_cost.py`(같은 비용), `combat_frontage.py`(전선 폭), `combat_rounds.py`(라운드 캡·슬롯 상한).

## 9. 군 규모 시스템 — 슬롯 상한 · 티어 · 라운드 캡 (**스팸 해결책, 확정**)

### 왜 (스팸 문제 해결)
§8에서 발견한 "싸고 많은 유닛이 유리" 문제를, **병력수를 상한으로 묶어** 해결. 병력수가 같으면 §7의 검증된 밸런스가 그대로 작동(→ `combat_rounds.py` 슬롯 테스트에서 모든 유닛이 명확한 카운터 보유 확인).

### 두 가지 캡 — ✅ **둘 다 구현**
1. **운용 부대 수 상한** (전략 층) — 맵에 동시 운용 가능한 부대 개수. **구현: 기본 3 + (성 레벨−1) + 연구 '군제 개편'(+1).** 이게 티어의 존재 이유(아래 측정 참조).
2. **부대당 병력 상한** (전술 층) — 한 부대 병력 수(슬롯). **구현: ARMY_CAP=20.** 전투를 "같은 병력수"로 만들어 밸런스 기반.
- **총 군사력 = 부대 수 × 부대당 병력**. 둘 다 성장으로 확대.

> **티어 밸런스 측정 결과(중요):** 동일 예산이면 T1 물량이 T2/T3를 전멸(랜체스터 제곱). **동일 슬롯이면 T3가 100% 승리.** → 티어는 **부대 수 상한이 있을 때만** 의미. 상한 구현으로 "성장한 경제 → 고티어로 슬롯 채우기 = 질적 우위"가 성립. **AI도 티어 성장(14턴마다 +1) + 지휘관 정예화 버프 + 대규모 원정대(본대 70% 투입)**로 대칭.
>
> **시작·수비 개편:** 스타터 부대 제거 → **주둔군 시작**(중갑5·창5·경기6). 초반 수입 상향(성 기본 수입 식/목/석 +1). **성 수비 버그 수정**: 주둔군은 부대가 성에 없어도 자동 수성(executeMove가 E→P 진입을 항상 defendCastle로 처리). 밸런스 시뮬레이션(몰입 봇): **플레이어 55% / 적 37% / 시간초과 8%, 평균 26.6턴.**

### 상한 성장 경로
- **성 레벨업** → 두 상한 확대 (기지 규모 = 병참).
- **연구(테크)** → 상한 확대 + 티어 해금.
- **영웅(지휘관)** → 붙은 부대의 **병력 상한 +보너스** (좋은 영웅 = 더 큰 군대). *무영웅 부대도 기본 상한은 있음.*

### 티어제 (병력 스펙 성장)
- 티어 = 6유닛의 **강화 레벨(스탯 배율)**. 유닛 6종 유지, 티어는 **배율 공식 하나로 관리** (6×9=54개 개별 유닛 지양).
- 고티어 = 강함 + 생산·자원 비용↑(특히 철). 제한된 슬롯을 고티어로 채워 강해짐 = **경제 → 질**.
- 첫 작: **3~5티어**로 시작, 9티어는 확장.
- 단순화: **1병력 = 1슬롯** (티어는 슬롯 소모 동일, 스탯만 상승).

### 라운드 캡 (전투 템포)
- 전투는 정해진 라운드(예: 6R)만 교전 후, 전멸 아니면 **잔존 전력으로 판정** → 빠른 해결, 재교전·후퇴 가능.
- 검증: 라운드 캡은 삼각을 깨지 않음(6R에서 보병>기병 1.00·기병>궁병 0.80·궁병>보병 1.00).

### 식량 유지비와의 관계 — ✅ **구현**
- **슬롯 상한 = 하드 천장** (스팸 차단·성장 곡선). **식량 유지비 = 소프트 비용** (천장까지 못 채울 수도). 두 층이 공존.
- **구현: 무료 유지 20병력, 초과분 1당 식량 0.2/턴.** 소규모(시작 16)는 무부담, 대군일수록 농장 투자 필요. 식량 고갈 시 그 턴 생산 중단.
- **석재 소비처 ✅**: 반복형 **성벽 보강**(석재↑ 비용, 수성 방어 +5%/레벨, 최대 +30%) — 후반 석재 잉여 해소.
- **영웅 성장 ✅**: 몬스터 처치 → **경험치 아이템**(사냥1/토벌2/레이드3) → 영웅 **등급 승급**(★1→2→3, 각 2·4개). 값싼 선술집 ★1을 키우는 성장 루트.
- **UX ✅**: 성 내정 패널을 탭(건물/연구/선술집/출전)으로 분리.

## 10. Phase B 상세 설계 (수직 슬라이스 MVP)

> 목표: 성장 → 편성 → 정복 루프가 처음부터 끝까지 한 바퀴 도는 최소 게임. 수치는 플레이스홀더로 시작해 플레이로 조율.

### 맵 — **노드형으로 시작 → 타일 그리드로 확장** — ✅ **타일 프로토타입 착수**
- **✅ game.js `setMap(mapDef)`**: 맵(NODES/EDGES/MONSTERS)을 주입 가능하게 리팩터링 → 로직 중복 없이 노드↔타일 교체. 기본맵=17노드.
- **✅ phaseC_tilemap.html**: 14×9 사각 그리드(game.js 재사용). 중앙 행=몬스터 없는 침공 회랑(P→E 11칸), 자원·둥지는 가지. 이동·전투·경제·저장 전부 재사용. (지형 효과·안개는 다음 단계.)

- **Phase B: 노드형.** 8~12개 거점(노드) — 성 2 · 자원지 · 몬스터 둥지 · 중계 노드 — 를 길(엣지)로 연결. 세력 2.
- **확장 경로**: 타일 그리드는 "타일=노드, 이웃=엣지"인 그래프의 특수형. 아래 추상화를 지키면 **로직 재작성 없이 맵 생성·렌더러만 교체**.
- **핵심 규율**: 게임 로직은 **(x,y) 좌표가 아니라 노드ID + 인접 + 이동비용의 '맵 추상화'에만 의존**. → 이동·길찾기·채집·전투·AI 그대로 재사용.
- 확장 시 새로 드는 일(추가지 재작성 아님): 그리드 렌더러 · 맵 생성/배치 · 타일 전용 요소(지형 영역·시야).
- **안개 없음** (Phase B 단순화).

### 이동
- 부대는 턴당 **이동력**(예: 3)만큼 **인접 노드로 이동** (엣지 비용 차감).
- 부대 = 노드 위의 스택(편성 + 선택적 영웅). §9의 **부대 수 상한 · 부대당 병력 상한** 적용(Phase B는 상한 고정, 성장 없음).

### 경제
- **채집**: 자원 노드에 부대 주둔 → 매 턴 해당 자원 획득. **내정 영웅** 있으면 채집량↑.
- **성 기본 수입**: 소량 자동.
- **생산**: 성에서 유닛 큐(자원 소모) → 성에서 스폰. **6유닛 전부 해금(테크 없음), 티어 T1 고정.**

### 전투·정복
- 적 부대가 있는 노드로 이동 = **자동 전투**(§7 엔진, 프로토타입 그대로).
- **성 공격 = 공성**: 성은 수비 보정(방어 버프 + 주둔군). **성 함락 = 승리 / 내 성 함락 = 패배.**

### 영웅
- 스타터 **내정 1 + 전투 1** 고정 (선술집·성장·재배치 없음).

### AI (스크립트형)
- 채집·생산 → 병력이 일정 규모가 되면 부대를 모아 플레이어 성으로 진군·공격. "똑똑함"보다 "돌아감" 우선.

### 턴 구조
- 플레이어 페이즈(이동·채집 지정·생산 큐) → 전투 해결 → AI 페이즈 → 자원 수입 정산.

### 시작 조건
- 각 세력: 성 1 + 스타터 부대 1 + 영웅 2(내정·전투) + 시작 자원.

### 구현 엔진
- **HTML/JS(브라우저)** — Phase A 프로토타입을 이어서 확장. 개인 개발 최저 마찰.

### Phase B에서 빼는 것 (재확인)
- 선술집·토벌·중앙 레이드·테크·성장 곡선·티어 상승·다수 세력·안개. → 모두 Phase B 검증 이후.

### 결정된 기본값 (변경 가능)
- **맵 = 노드형**(그래프 추상화로 타일 확장 대비) · 안개 OFF · 티어 T1 고정 · JS 브라우저 엔진.
- 최종 목표 맵은 **타일 그리드**(사각) — Phase B 루프 검증 후 확장.

## 11. 개발 하네스 (`mini4x/`)

전투 엔진 복제(드리프트)를 막기 위한 단일 소스 도구 모음.

- **`engine.js`** — 단일 캔노니컬 전투 엔진(node+브라우저). §7 규칙 구현. **밸런스 수치는 여기서만 수정.**
- **`verify.js`** — 자동 검증 하네스. `node verify.js` → 삼각·지배유닛·카운터·전열분산·티어 (현재 **12 PASS / 0 FAIL**).
- **`build.js`** — `engine.js`를 HTML 템플릿에 인라인 → 자체 완결 프로토타입 생성(복제 제거).
- **워크플로**: engine.js 수정 → `node verify.js`(밸런스 확인) → `node build.js`(프로토타입 갱신).
- 확장 예정: `mapgraph.js`(노드 그래프+Dijkstra, 타일 확장 시 `buildTileMap()`만 교체).
- **✅ 저장/불러오기**: 상태 JSON 직렬화 → localStorage 자동 저장(매 턴) + 파일 내보내기/가져오기. 30~60분 세션 대응. 숨은 상태(NODES.ANCIENT.owner)도 함께 저장, 깊은 복사 스냅샷.
